# Release Notes

Version 1.2.1 introduces several quality-of-life improvements to patient management, enhancing data entry flexibility, improving search robustness, extending character limits for clinical tracking, and refining the PDF report experience.

---

# Backend

## Modifications

### DNI Flexibility
The Patient ID (DNI) field constraint has been relaxed to better support diverse patient intake scenarios.
- **Optional DNI:** The DNI is no longer a required field (`NOT NULL` constraints removed) when creating or updating a patient.
- **Non-Unique DNI:** The strict unique constraint (`UNIQUE`) on the DNI column has been lifted to prevent database integrity errors in edge cases (e.g., duplicated placeholder values).
- **Validation DTOs:** Validation rules within `PatientCreateDto` and `PatientUpdateDto` have been adjusted to reflect these new optional constraints. 
- **Tests Updated:** `PatientControllerTest` logic has been adapted to handle the removed DNI validations correctly.

---

# Frontend

## Improvements and Refactoring

### Patient Search Enhancements
The patient directory search function has been made significantly more robust.
- **Null Reference Handlers:** `applyFilters` logic now gracefully utilizes fallback strings when encountering null fields (such as missing DNI, email, or surnames), preventing application crashes during typed searches.

### Character Limit Expansion
Character constraints for clinical input have been massively expanded to support comprehensive patient logging.
- **Notes and Tracings:** The hardcoded `255` character limit barrier for "Valoraciones" (Notes) and "Seguimiento" (Tracings) has been increased to `10000` characters across both the `patient-detail` component TS definitions and HTML templates.

### Automated Descending Sort
The UI now naturally surfaces the most recent clinical interactions faster.
- **Valoraciones & Seguimiento Order:** Clinical records, notes, and tracings are now automatically sorted in descending order (by `createdAt`), so the latest entries immediately appear at the top of the timeline upon creation or editing.

### PDF Engine Tweaks
The PDF report logic has been adjusted for usability and visual layout.
- **Section Reordering:** The "SEGUIMIENTO" module has been pushed down to immediately follow "TRATAMIENTO", supported by a distinct separator line to visually group active clinical tracking.
- **In-Browser Preview:** The PDF generator action has been retooled from an automatic file download into an in-browser preview functionality. The button has been renamed from "Descargar PDF" to "Vista previa", opening the resulting `bloburl` in a new browser tab for immediate reading. Email attachment generation continues to function as before.
