# Release Notes

Version 1.2.2 delivers significant improvements across patients, PDF reports, and UI/UX. Highlights include adding a second last name field to the patient model, a redesigned PDF layout with clinic branding and dynamic content filtering, normalized search, background tab opening from the patient list, and full-width modals for clinical history, antecedents, and notes.

---

# Backend

## New Features

### Second Last Name (`secondLastName`)
Full-stack addition of an optional second last name field for patients.
- **Model:** New `secondLastName` column (`@Column(name = "second_last_name")`) added to the `Patient` entity.
- **DTOs:** `PatientCreateDto`, `PatientUpdateDto`, `PatientDto`, and `PatientListDto` updated to include `secondLastName` with `@Size(max = 50)` validation.
- **Service:** `PatientService` now maps `secondLastName` in create and update operations.
- **Tests:** `PatientControllerTest` and `PatientServiceTest` updated to cover the new field.

---

# Frontend

## New Features

### Open Patient in Background Tab
Patients in the main list now have an action button to **open the patient detail in a new browser tab** in the background (without stealing focus). Uses `Ctrl+Click` / `window.open()` with fallback focus management.

### Second Last Name Field
- **Model:** `Patient` TypeScript model extended with optional `secondLastName` property.
- **Patient List:** The patient table now displays the full name including the second last name.
- **Patient Detail:** Displayed alongside name and first last name.
- **Create/Edit Forms:** New `secondLastName` input field added in patient creation (main list) and patient edit (detail view) forms, laid out in a three-column row alongside name and first last name.
- **Search:** The patient search filter now includes `secondLastName` in its matching logic with null-safe handling.

## Improvements

### Normalized Patient Search
The patient search function now **normalizes accents and is case-insensitive**. Searching "garcia" will match "García", "GARCÍA", etc. Applied via Unicode normalization (`NFD` + diacritics removal).

### Full-Width Modals
All modals for clinical history, antecedents, notes, valoraciones/seguimiento, and content popups now expand to **full container width** (matching the patient details section at `1400px`) instead of the previous narrow 500px/700px. This provides a more spacious editing and viewing experience.

### PDF Report Redesign
Major overhaul of the PDF generation engine:
- **Header:** Clinic logo with contact information (web and email) displayed alongside the logo.
- **Footer:** Doctor name and professional qualifications (smaller font) in the footer of every page.
- **Title removed:** The previous "INFORME" title has been removed for a cleaner layout.
- **Dynamic Content Filtering:** Empty fields and sections are now automatically hidden from the generated PDF. Uninformed patient data, empty antecedent fields, and blank clinical history sections are omitted entirely, producing clean and concise reports.
- **Spacing & Borders:** Corrected spacing, borders, and pagination behavior in `jspdf-autotable` for consistent table rendering across pages.

### UI/UX Polish
- **"No proporcionado" indicators:** Patient detail fields that have no value now display "No proporcionado" in *italic* style, making it visually clear which data is missing vs. intentionally blank.
