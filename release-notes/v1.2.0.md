# Release Notes 

Version 1.2.0 introduces the new "Seguimiento" module, fully integrated into the medical record flow, architectural improvements to the clinical history view, and a significant redesign of the PDF report structure to prioritize general patient background.

---

# Backend

## New Features

### Tracing (Seguimiento) System

A new entity has been introduced to track the patient's progress over time, linked directly to each medical record.

- **Dedicated REST sub-resource:**  
  `/api/medical-records/{id}/tracings`
- **Full CRUD operations:** Create, read, update, and delete support.
- **Persistence Layer:** New `Tracing` entity with automated `createdAt` auditing.
- **DTO & Mapping:** Full integration with `MedicalRecordDto` and `MedicalRecordCreateDto`.
- **Testing:** Comprehensive unit and integration test suite for the new controller and service layers.

### Structural Cleanup

- **Entity Removal:** The legacy `Diagnose` entity and its associated tables have been removed. Diagnoses, protocols, and prescriptions are now stored directly as optimized fields within the `MedicalRecord` entity for improved performance and a more intuitive data model.

---

# Frontend

## New Features

### "Seguimiento" (Tracing) Interface

The "Seguimiento" module has been implemented with functional parity to the "Valoraciones" (Notes) system to ensure a consistent user experience.

- **Integrated Workflow:** Seamlessly added to the "Historia Clínica" edit/view modals, appearing after the Treatment section.
- **Visual Consistency:** Implemented with the same design tokens and layout as the evaluation system.
- **Inline Editing:** Quick-add textarea with real-time validation and character counting.
- **Interactive List:** Support for pagination, "read more" popups for long descriptions, and immediate local updates.

---

## Improvements and Refactoring

### PDF Report Redesign

The PDF generation engine has been extensively updated based on professional clinical reporting standards:

- **Information Hierarchy:** The "Antecedentes Generales" section (previously "Antecedentes Médicos") has been moved to the top of the report, prioritizing structural patient information.
- **Clinical Clarity:** Renamed sections to more professional terminology (e.g., "ANTECEDENTES GENERALES").
- **Layout Optimization:**  
  - Removed unnecessary blank rows separating diagnoses for a cleaner look.
  - Added the "SEGUIMIENTO" section immediately before "VALORACIONES".
  - Refined treatment ("TRATAMIENTO") rendering to emphasize prescriptions even without a formal diagnosis.
- **Deterministic Data Loading:** Implemented sequential `forkJoin` fetching to ensure all sub-resources (notes, tracings, files) are fully loaded before PDF generation.

### Shared Modal Architecture

Optimized the modal system to handle multiple sub-resource types efficiently:

- **Context-Aware UI:** The shared editing modal now dynamically updates headers, placeholders, and action buttons based on whether it's handling a "Valoración" or a "Seguimiento".
- **Safety Dialogs:** Added standardized `confirm()` prompts for deleting records within both sections to prevent accidental data loss.
- **Empty State Normalization:** Unified the appearance of sections when no data is present ("Sin seguimiento", "Sin valoraciones").

## Bug Fixes

### Data Synchronization

- **ID Handling:** Fixed temporary ID generation for new items to prevent collisions before backend persistence.
- **Form Resets:** Ensured "Seguimiento" state is properly cleared when switching between different medical records or closing modals.
- **Validation Consistency:** Standardized minimum character requirements and error messaging between clinical notes and progress tracings.
