# Release Notes

Version 1.3.0 introduces a new template-based consent PDF generation flow on the frontend, including dynamic form rendering, selectable document templates, and support for pre-designed Spanish consent forms with field-level placement.

---

# Backend

## Modifications

No backend changes are included in this release.

---

# Frontend

## New Features

### Template-Based PDF Generator
A new PDF generation path has been added alongside the existing clinical history exporter.

- **Dual flow selector:** Users can now choose between `Historial clínico` (legacy) and template-based documents from the same PDF modal.
- **Template engine (`pdf-lib`):** Added a new `PdfManagerService` flow that fills existing PDF assets by writing text into configured coordinates.
- **Dynamic template forms:** The UI now builds form controls from template metadata (`fields`) at runtime, including required validations and max length rules.
- **Per-field input types:** Template fields now support both text inputs and select controls.

### New Consent Templates (ES)
The following consent templates are now integrated and available in the selector:

- **Consentimiento Infusión de Vitamina C** (`vitc_infusion_es`)
- **Consentimiento básico sin fármacos** (`non_pharma_es`)
- **Consentimiento Autohemotransfusión con Ozono** (`ozone_autohemo_es`)

Each template includes mapped blank fields for patient identity data, legal representative data, date, and treatment-specific blanks.

## Improvements and Refactoring

### PDF UX and Component Structure

- **New reusable selector component:** `PdfTemplateSelectorComponent` encapsulates document type selection and styling.
- **Patients module wiring:** Added template selector declarations/import usage in the patients feature module.
- **Auto-prefill in template flow:** Patient `fullName`, `dni`, computed `age`, and current `date` are now preloaded when available.
- **Typed field metadata:** Template field model expanded to support `inputType` and `options` for configurable form rendering.

### Non-Pharma Consent Alignment

Field placement for `non_pharma_es` was calibrated to align with actual blank lines in the source PDF, including:

- `Antecedentes generales`
- `Antecedentes relacionados`
- `Procedimiento a realizar`

## Bug Fixes

### PDF Blob Compatibility

- Fixed Blob creation typing issues when saving generated template PDFs by normalizing saved bytes before creating the Blob payload.
